SKILL.md Summary

PHP MCP Server Generator

このスキルは、PHP公式 SDK を使った MCP サーバー一式を自動生成するための設計書です。 単にファイルを並べるのではなく、Tools / Resources / Prompts / Testsまで含めた 「そのまま開発を始められる構成」を作ることが主目的です。

このスキルがやること

  1. ユーザーからプロジェクト要件を聞き出す
  2. PHP 8.2+ 前提の MCP サーバープロジェクトを生成する
  3. ツール・リソース・プロンプト・テストをひと通り揃える
  4. そのまま実行・テスト・Claude Desktop 接続までできる形にする
6

最初にユーザーへ確認する要件

4

主な生成対象: Tools / Resources / Prompts / Tests

8.2+

必須 PHP バージョン

PSR-12

コード規約の基本方針

McpTool McpResource McpResourceTemplate McpPrompt PHPUnit PSR-16 Cache

何が揃うか

ファイル 役割
composer.json 依存関係、autoload、PHP バージョン定義
server.php MCP サーバー起動エントリーポイント
src/Tools/* MCP Tool の実装
src/Resources/* 固定/動的 Resource の実装
src/Prompts/* Prompt 生成ロジック
tests/* PHPUnit による Tool テスト

3. 重要テンプレートの読みどころ

composer.json

MCP SDK と PHPUnit を入れ、App\ / Tests\ の PSR-4 オートロードを設定します。

{
  "require": {
    "php": "^8.2",
    "mcp/sdk": "^0.1"
  },
  "require-dev": {
    "phpunit/phpunit": "^10.0",
    "symfony/cache": "^6.4"
  }
}

server.php

Discovery + Cache を有効にし、最終的に StdioTransport で実行します。

$server = Server::builder()
    ->setServerInfo('{Project Name}', '1.0.0')
    ->setDiscovery(...)
    ->build();

$transport = new StdioTransport();
$server->run($transport);

Tool 実装

#[McpTool] 属性を使ってメソッドを公開し、必要なら #[Schema] で入力制約を付けます。

#[McpTool]
public function greet(string $name): string
{
    return "Hello, {$name}!";
}

Resource 実装

固定 URI には #[McpResource]、可変 URI には #[McpResourceTemplate] を使います。

#[McpResource(uri: 'config://settings')]
public function getSettings(): array
{
    return ['key' => 'value'];
}

Prompt 実装

#[McpPrompt] で会話配列を返し、候補値がある場合は #[CompletionProvider] を使います。

#[McpPrompt(name: 'code_review')]
public function reviewCode(string $language, string $code): array
{
    return [['role' => 'user', 'content' => '...']];
}

テスト

少なくとも Tool は PHPUnit で正常系・異常系を押さえる前提です。

public function testCalculateDivideByZero(): void
{
    $this->expectException(\InvalidArgumentException::class);
    $this->tool->performCalculation(10, 0, 'divide');
}

4. 実装ルール

必須ルール

  • PHP Attribute ベースで実装する
  • 全ファイルで declare(strict_types=1);
  • PSR-12 に従う
  • 入力バリデーションは #[Schema] で表現する

品質ルール

  • 例外は具体的に投げる
  • Tool には PHPUnit テストを用意する
  • PHPDoc を付ける
  • 本番向けには Discovery Cache を使う

5. パターン集

Simple Tool

#[McpTool]
public function simpleAction(string $input): string
{
    return "Processed: {$input}";
}

Validation 付き Tool

#[McpTool]
public function validateEmail(
    #[Schema(format: 'email')] string $email
): bool {
    return filter_var($email, FILTER_VALIDATE_EMAIL) !== false;
}

Enum を受ける Tool

enum Status: string {
    case ACTIVE = 'active';
    case INACTIVE = 'inactive';
}

Static Resource

#[McpResource(uri: 'config://settings')]
public function getSettings(): array

Dynamic Resource

#[McpResourceTemplate(uriTemplate: 'user://{id}')]
public function getUser(string $id): array

Prompt

#[McpPrompt]
public function generateDocs(string $code): array
